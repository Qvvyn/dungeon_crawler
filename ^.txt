Plan: Ambush crash fix + Late-game stability + Sector lighting                                                 │
│                                                                                                                │
│ Context                                                                                                        │
│                                                                                                                │
│ Three things, in priority order:                                                                               │
│ 0. Urgent bug — the ambush spawns enemies inside a physics body_entered callback, which                        │
│    Godot forbids ("Can't change this state while flushing queries") → crash. Must defer.                       │
│ 1. T100+ stability (the user's flagged crash) — the initial floor is already capped at 35                      │
│ (MAX_DUNGEON_ENEMIES, themed rooms counted), so the explosion comes from runtime                               │
│ spawners that ignore the cap, plus loot bags piling up during fast late-game kills.                            │
│ 2. Sector lighting — DOOM atmosphere: torch flicker + per-room darkness levels + a few                         │
│ strobe/dark rooms. Uniquely effective because the ASCII shader is brightness-driven (dim                       │
│ areas literally thin into sparse glyphs).                                                                      │
│                                                                                                                │
│ Out of scope (note for later): item/damage rebalance, 2D top-down vision radius, and a shader                  │
│ "blinded" vignette — bigger separate efforts.                                                                  │
│                                                                                                                │
│ ---                                                                                                            │
│ Part 0 — Fix the ambush physics-flush crash (scripts/AmbushController.gd)                                      │
│                                                                                                                │
│ _on_tripwire runs during physics flush; _arm() → spawn_ambush_wave() → add_child of an                         │
│ enemy with collision = illegal mid-flush.                                                                      │
│ - Add an ARMING state to the enum.                                                                             │
│ - _on_tripwire: on valid trip, set _state = ARMING and call_deferred("_arm") (don't call                       │
│ _arm directly).                                                                                                │
│ - _arm: do seal/close + wave spawn + text/sound, then set _state = ACTIVE at the end.                          │
│ - _process: only run the clear-check when _state == ACTIVE (ARMING is ignored), which also                     │
│ avoids a race where an empty _wave triggers an instant _clear() before the wave exists.                        │
│                                                                                                                │
│ ---                                                                                                            │
│ Part 1 — Late-game stability                                                                                   │
│                                                                                                                │
│ 1a. Global live-enemy cap on ALL runtime spawners                                                              │
│                                                                                                                │
│ The initial pass already caps at 35. The leak is runtime spawns that don't check live count.                   │
│ Add one shared guard and apply it everywhere a new enemy is born at runtime.                                   │
│ - Add World.can_spawn_enemy() -> bool → get_tree().get_nodes_in_group("enemy").size() < MAX_LIVE_ENEMIES (new const, e.g. 45 — above the 35 build cap so combat can breathe, but bounded).                                   │
│ - Guard these spawn sites (call the helper / inline the count check before spawning):                          │
│   - EnemyBase._do_split() (splitter clones) — skip splitting if at cap.                                        │
│   - EnemyBase._schedule_catacombs_reanimation() timer callback (zombie revive) — skip if at cap.               │
│   - EnemySummoner._try_summon() — already self-limits to live < MAX_MINIONS (3); leave as is                   │
│ (or align to the shared helper for consistency).                                                               │
│   - The per-frame dungeon spawn-queue drain (_process drain of _dungeon_spawn_queue) — stop                    │
│ draining when at cap (it resumes as enemies die). Find the drain loop near                                     │
│ _dungeon_spawn_queue.pop_front().                                                                              │
│ - Result: total live enemies never exceeds ~45 regardless of source.                                           │
│                                                                                                                │
│ 1b. Live loot-bag cap + consolidation                                                                          │
│                                                                                                                │
│ Bags spawn faster than the player collects them in late-game; they accumulate unbounded.                       │
│ - Track via the existing "loot_bag" group. Add World.enforce_loot_cap() called whenever a bag                  │
│ is added (or on a cheap periodic timer in _process).                                                           │
│ - When live bags exceed MAX_LIVE_LOOT_BAGS (e.g. 12), merge the oldest overflow bags into a                    │
│ single mega-bag, reusing the existing room-clear merge (the MEGA_BAG_MAX_ITEMS = 40 +                          │
│ overflow→gold logic in World.gd, ~L3636). Net effect: bounded bag nodes, items preserved up                    │
│ to 40, the rest become gold (consistent with current mega-bag behavior).                                       │
│ - Simpler fallback if merge reuse is awkward: convert the oldest overflow bags' items to gold                  │
│ (sum sell_value → GameState.gold) and queue_free them.                                                         │
│                                                                                                                │
│ 1c. (cheap) trim per-kill drop churn                                                                           │
│                                                                                                                │
│ Drop chance already shrinks with difficulty; the cap fixes most. Optionally lower                              │
│ LootBag item count at the very top (e.g. cap to 1–2 items when difficulty is high) to cut                      │
│ item-generation cost. Keep minimal.                                                                            │
│                                                                                                                │
│ ---                                                                                                            │
│ Part 2 — Sector lighting (FP)                                                                                  │
│                                                                                                                │
│ 2a. Torch flicker (FirstPersonRig.gd)                                                                          │
│                                                                                                                │
│ In _process, modulate _player_light.light_energy with a subtle layered sine flicker                            │
│ (mirror the existing pulse pattern at ~L1074: sin(Time.get_ticks_msec()*k)*0.5+0.5). Small                     │
│ amplitude so it reads as a living torch, not a strobe. Respect GameState.disable_flashing                      │
│ (reduce amplitude when set).                                                                                   │
│                                                                                                                │
│ 2b. Per-room darkness levels                                                                                   │
│                                                                                                                │
│ - Store the Environment as a member (_env) in _build_scene so ambient_light_energy is                          │
│ tweakable per-frame (currently a local).                                                                       │
│ - World assigns each room a light level: a parallel Array[float] (or small enum: NORMAL / DARK                 │
│ / FLICKER) sized to _rooms. Most rooms NORMAL; a minority DARK (torch-only) and FLICKER.                       │
│ Pass to the rig via a new set_room_lighting(rooms: Array, levels: Array) after the grid is set                 │
│ (alongside set_grid).                                                                                          │
│ - In the rig _process: find the player's current room (scan _rooms for Rect2i.has_point,                       │
│ cached — only rescan when the player leaves the cached rect). Lerp _env.ambient_light_energy                   │
│ (and optionally _player_light.omni_range) toward the current room's target. DARK rooms drop                    │
│ ambient near 0 so only the torch lights a small pool — the brightness-driven shader then                       │
│ renders the room edges as sparse ./: glyphs fading to black.                                                   │
│                                                                                                                │
│ 2c. Flicker/strobe rooms                                                                                       │
│                                                                                                                │
│ For rooms flagged FLICKER, drive ambient with an erratic flicker (e.g. layered sines + occasional              │
│ dropouts) instead of a steady level — DOOM's broken-light dread. Gate amplitude on                             │
│ GameState.disable_flashing.                                                                                    │
│                                                                                                                │
│ ---                                                                                                            │
│ Files                                                                                                          │
│                                                                                                                │
│ - scripts/AmbushController.gd — defer arm (Part 0).                                                            │
│ - scripts/World.gd — MAX_LIVE_ENEMIES/can_spawn_enemy, queue-drain cap, loot-bag cap +                         │
│ consolidation, per-room light levels + set_room_lighting call.                                                 │
│ - scripts/EnemyBase.gd — cap guards in _do_split + catacombs reanimation.                                      │
│ - scripts/FirstPersonRig.gd — torch flicker, _env member, set_room_lighting, per-room                          │
│ ambient drive.                                                                                                 │
│ - (no shader edit this pass — vignette/blinded deferred.)                                                      │
│                                                                                                                │
│ Verification                                                                                                   │
│                                                                                                                │
│ 1. Ambush: walk into an ambush room — no console error; walls seal, wave spawns a frame later,                 │
│ clears reopen. (Fixes the reported area_set_shape_disabled crash.)                                             │
│ 2. Stability: at high difficulty, fight in a churny room — live enemy count stays ≤ ~45                        │
│ (check get_nodes_in_group("enemy").size()), splitters/zombies stop adding at the cap, and                      │
│ loot bags never exceed ~12 (overflow merges to a mega-bag / gold). No slowdown/crash.                          │
│ 3. Lighting: torch subtly flickers; walking into a DARK room dims it to a torch-lit pool (sparse               │
│ glyphs at the edges); a FLICKER room visibly strobes. disable_flashing tames amplitudes.                       │
│ 4. No regressions: normal floors still populate; loot still collectable; FP renders correctly.
